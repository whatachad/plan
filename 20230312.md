[ 페이지 별 논의가 필요한 사항 ]

1. 🏠홈 화면
1-1. 네비게이션 바 : 🏠홈 / 🏋️‍♀️운동시설 / 📅자기관리 / ⚙️마이페이지 / (한글영문전환 셀렉터)
1-2. 글로벌스타일
- 서체 크기 (로고, 대제목, 소제목, 본문, 서브)
- 컬러 (액티브, 서체, 보더, 백그라운드...)
- 여백 (마진,패딩, 갭)

2. 🏋️‍♀️운동설비 페이지
2-1. 🏋️‍♀️리스트 화면
- 검색바
- 무한 스크롤
- 내가 찜한 운동설비 목록

2-2. 🗺️지도화면 (카카오 지도 api)
- 내 위치 주변의 운동설비를 표시.
- 내 위치 재설정 기능.
- 운동설비의 타입 별 탭 (전체/헬스장/운동장 ...).
- 내가 찜한 설비 목록만 보기.

2-3. 운동설비 디테일
- 서버에서 받아올 데이터 : id / 이미지 / 설명 / 위치(위도경도,한글주소("oo도 oo시 oo구 oo동")) / 설비타입(체육관, 운동장...)
- 클라이언트에서 처리해야 할 데이터 : 내 위치로부터의 거리(km)
- 유저별 서버에서 받아올 데이터 : 즐겨찾기(찜) 유무

2-4. 운동설비 추가/수정 화면(어드민 페이지가 있을 시)

3. 📅 자기관리 페이지 (로그인필요)
3-1. 📅달력화면(월별)
- 화면구성 : 달력, 투두리스트
- 달력의 날짜에는 그 날 발생한 금액변화(예) -20,000원), 투두리스트가 기재됨
- 예산보다 더 큰 지출이 발생한 날은 빨간색 스타일로 표기.
- 날짜 클릭 시, 일별 디테일 화면으로 이동.

3-2. 📜월별 리스트 화면
- 수입지출이체 리스트, 투두리스트
- 예산보다 더 큰 지출이 발생한 일은
- 항목 추가, 제거 버튼.

3-3. 일별 디테일화면
- 오늘로 이동 버튼.
- 항목 추가, 제거버튼.

3-3. 항목 작성/수정 화면 (자기관리 앱으로서, 어디까지 구현해야할지 논의 필요)
3-3-1. 가계부 데이터
- 카테고리: ["식비", "교통", ...] 셀렉터
- 타입 : ["수입","지출","이체"] 셀렉터
- 날짜선택 : 날짜타입
- 내용 : string
- 금액 : number

3-3-2. 투두리스트 데이터
- 제목 : string
- 내용 : string
- 상태 : boolean
- 컬러 : string
- 날짜선택 : 날짜타입
